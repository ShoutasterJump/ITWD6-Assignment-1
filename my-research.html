Copyright, Creative Common (CC) Licenses, Fair Use

  1. Basic Principles of Copyright
    • Copyright provides protection to original works of authorship such as literature, music, art, and software.
    • It grants the creator exclusive rights to reproduce, distribute, and display the work.
    • Copyright is automatically granted as soon as the work is created, no registration is required.
    • The typical duration of copyright lasts for the lifetime of the creator plus 50 years in New Zealand.
    • For more details: Copyright Council of New Zealand
  2. Creative Commons (CC) Licenses
    • CC BY: This license lets others distribute, remix, adapt, and build upon your work, even commercially, as long as they credit you for the original creation.
    • CC BY-SA: This license allows others to remix, adapt, and build upon your work even for commercial purposes, as long as they credit you and license their new creations under the same terms.
    • CC BY-ND: This license allows for redistribution, commercial and non-commercial, as long as it is passed along unchanged and in whole, with credit to the original creator.
    • CC BY-NC: This license lets others remix, adapt, and build upon your work non-commercially, and although their new works must acknowledge you and be non-commercial, they don’t have to license their derivative works on the same terms.
    • For more details: Creative Commons Licenses
  3. Fair Use in New Zealand Law
    • New Zealand law does not have a "fair use" provision like the United States, but it has "fair dealing" provisions.
    • Fair dealing allows limited use of copyrighted material for specific purposes such as criticism, review, and reporting current events.
    • Fair dealing provisions are much narrower than fair use, allowing exceptions for research, private study, or news reporting.
    • For more details: Copyright Act 1994

Privacy & Web Privacy Policy

What is a Privacy Policy?

A privacy policy is a statement or legal document that outlines how an organization collects, uses, manages, and protects the personal information of its users. It is designed to inform individuals about their data rights and the measures taken to ensure data security and compliance with privacy laws.

13 Principles of Privacy

  1. Accountability: Organizations must take responsibility for the personal data they hold and ensure it is handled in compliance with privacy laws.
  2. Identifiable Information: The collection of personal data should be limited to what is necessary for the intended purpose and should not be used for purposes beyond those for which it was collected.
  3. Consent: Organizations must obtain explicit consent from individuals before collecting or processing their personal data.
  4. Purpose Limitation: Personal data should only be collected for specific, legitimate purposes and not used in ways that are incompatible with those purposes.
  5. Data Minimization: The amount of personal data collected should be limited to what is necessary for the intended purpose.
  6. Accuracy: Organizations should ensure that personal data is accurate and kept up to date.
  7. Retention: Personal data should not be kept for longer than necessary to fulfill the purpose for which it was collected.
  8. Security: Adequate measures should be taken to protect personal data from unauthorized access, disclosure, or destruction.
  9. Access: Individuals have the right to access their personal data and request corrections or deletions.
  10. Transparency: Organizations should be transparent about their data practices and provide clear information about how personal data is used.
  11. Accountability: Organizations should be accountable for their data practices and be prepared to demonstrate compliance with privacy principles.
  12. Third-Party Transfers: If personal data is shared with third parties, organizations must ensure those parties also comply with privacy principles.
  13. Regulatory Compliance: Organizations must comply with relevant privacy laws and regulations applicable to their operations.

For more detailed information on these principles, visit Privacy International's overview of privacy principles.

WEB PERFORMANCE AND MAINTENANCE

Ways to Reduce Page Loading Time

  1. Optimize Images: Compress and resize images to reduce their file size without sacrificing quality. Tools like TinyPNG can help with this.
  2. Minimize HTTP Requests: Reduce the number of elements on your page (such as images, scripts, and CSS files) to decrease the number of HTTP requests. Combine CSS and JavaScript files where possible.
  3. Enable Browser Caching: Use caching to store frequently accessed resources locally in the user's browser. This allows for faster loading times on subsequent visits. Learn more about browser caching.
  4. Use a Content Delivery Network (CDN): Distribute your content across multiple servers worldwide to speed up delivery to users based on their geographic location. Services like Cloudflare offer CDN solutions.
  5. Minify CSS, JavaScript, and HTML: Remove unnecessary whitespace, comments, and code from these files to reduce their size. Tools like UglifyJS for JavaScript and CSSNano for CSS can be used for minification.

Important Tasks for Website Maintenance

  1. Regular Backups: Ensure that you regularly back up your website's data and files. This helps to recover your site quickly in case of data loss or security breaches. Tools like UpdraftPlus can automate this process.
  2. Update Software and Plugins: Keep your website's software, plugins, and themes up to date to protect against security vulnerabilities and improve functionality. WordPress provides guidance on updating software.
  3. Check for Broken Links: Regularly test and fix broken links to ensure a smooth user experience and maintain SEO rankings. Tools like Broken Link Checker can help identify these issues.
  4. Monitor Performance: Use tools to regularly check your website's performance, including load times and uptime. Google PageSpeed Insights can provide insights into performance improvements.
  5. Security Scans: Conduct regular security scans to identify and address vulnerabilities. Services like Sucuri offer comprehensive website security solutions.

WEB SECURITY

  1. Phishing: A deceptive attempt to steal sensitive information by masquerading as a trustworthy entity in electronic communications.
    • Preventive Measures: Use email filtering solutions, educate users about recognizing phishing attempts, and enable multi-factor authentication (MFA).
      • Learn more about Phishing
  2. Malware: Malicious software designed to damage, disrupt, or gain unauthorized access to systems.
    • Preventive Measures: Install antivirus software, keep software updated, and avoid downloading files from untrusted sources.
      • Learn more about Malware
  3. SQL Injection: An attack that allows hackers to interfere with the queries that an application makes to its database.
    • Preventive Measures: Use parameterized queries, validate user inputs, and apply database access controls.
      • Learn more about SQL Injection
  4. Denial of Service (DoS): An attack aimed at making a machine or network resource unavailable to its intended users by overwhelming it with a flood of requests.
    • Preventive Measures: Use a content delivery network (CDN), set up web application firewalls (WAFs), and implement rate-limiting.
      • Learn more about DoS
  5. Man-in-the-Middle (MITM): An attack where the attacker secretly intercepts and possibly alters the communication between two parties.
    • Preventive Measures: Use encryption (SSL/TLS), enable VPN for secure connections, and implement secure key exchange methods.
      • Learn more about MITM
